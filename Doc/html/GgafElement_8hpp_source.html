00007 #define GGAF_SAYONARA_DELAY (60*15)
00008 //GGAF_SAYONARA_DELAYは全Element共通の解放遅延フレーム数で、
00009 //長めに設定しないと、いろいろ不都合が多い。
00010 //消失後もなお参照が可能とするための仕組みで、
00011 //例えば、消滅後の爆発SE等で、3D効果のためSE発声元座標情報がしばらく必要とか、
00012 //消滅後の消滅アクターに属する子アクターの発射弾が、親アクターの座標を参照しているなど、
00013 //消滅しても、しばらくは残存させなければいけない場合がある。
00014 //基本的に、消滅確定→完全に消失までのフレームの最大フレームを指定する。
00015 //GGAF_SAYONARA_DELAYを短めに設定すると、現在は10秒(60*15)
00016 //消滅後の子の発射弾については、残された発射弾が突然消えるので少し不自然。という程度で問題はない。
00017 //しかし、SEの3D効果は、エラーになるため、SE再生時間の最大フレーム数を設定しなければいけない。
00018 //例えば爆発SEが１分かかるような物が一つでも使用したい場合は、最大の1分(60*60)を設定しなければならない。

00569     // ＜メモ initialize(), onReset(), onActive() の使い分けについて＞
00570     // 状態をリセット処理は、どこに実装するべきか？
00571     // initialize() ・・・ 生成後、最初の１回だけ事前にコールバックされる。初期化処理等に使用することを想定。
00572     // onReset()    ・・・ 生成後、initialize() の後、１回だけ無条件でコールされる。その後、reset() 実行ででコールバックされる。
00573     // onActive()   ・・・ 生成後、initialize() の後の活動状態時 １フレーム目の behave() 直前にコールされる。
00574     //                     その後 非活動→活動状態になった時、自動的にコールバックされる。
00575     // ノード(アクターやシーン)の処理サイクルによって使い分ける。
00576     // アクター生成後、そのコンストラクタで何もしなかった場合、最初の nextFrame() 時に
00577     // initialize() ＞ onReset() ＞onActive() の順に実行されることになる。
00578     //
00579     // ・１サイクルなノード(アクターやシーン)の場合
00580     // 生成後、初めての「終了」という状態を迎えた際に、そのまま解放となる場合。つまり、インスタンスを使いまわさない場合。
00581     // initialize(), onReset(), onActive() は事前に各１回実行されるので、どこに状態をリセット処理を実装しても差異は無いはず。
00582     // 多くの通常のアクターや、一時的な短いシーンはこれに該当すると予想する。
00583     // どこに実装しても良いが、initialize() で実装を統一したほうが、他のオブジェクトから操作しやすい。
00584     //
00585     // ・onActive() を使用すべき場合
00586     // 生成されたノードに、「表示」「非表示」という状態が存在し、使いまわす場合。
00587     // 表示(活動) 〜 非表示(非活動) が、ノードの１サイクルのと一致する場合。
00588     // initialize() ・・・ 初期化処理、
00589     // onActive()   ・・・ 状態リセット処理
00590     // と使い分けて記述し。 inacivate()、active() で状態リセット行う。
00591     // なぜ onReset() に状態リセット処理を記述し、reset()コールではいけないのかというと、
